# УМКарта

## Что реализовано
- Личный кабинет: регистрация, вход, выход.
- Просмотр карт только для авторизованных пользователей.
- Лимит после регистрации: 3 карты бесплатно, дальше нужна подписка.
- Тарифы с кнопками покупки (провайдер платежей: ЮKassa).
- Роли: `user`, `admin`.
- Админка доступна только администратору через личный кабинет.
- Админка умеет добавлять и удалять карты по `предмет -> класс -> параграф`.
- Поддержка импорта карт: `txt`, `xmind`, `xmmap`, `pdf`.
- Интерактивная mind-map визуализация + полноэкранный режим карты.
- Масштаб и перемещение карты: колесо мыши (`zoom`) и зажатая ЛКМ (`pan`).
- AI-нормализация карт при загрузке через админку (Kie.ai Gemini) с fallback на исходную структуру.

## Демо-админ
При первом запуске сервер автоматически создаёт администратора:
- email: `admin@umkarta.local`
- пароль: `admin12345`

Изменить можно через переменные окружения:
- `ADMIN_EMAIL`
- `ADMIN_PASSWORD`

## ЮKassa
Для реальных платежей заполните в `.env`:
- `YOOKASSA_SHOP_ID`
- `YOOKASSA_SECRET_KEY`

Webhook (в ЮKassa):
- `POST /api/yookassa/webhook`
- события: `payment.succeeded`, `payment.canceled` (и другие статусы, если придут)

Как работает логика подписки:
- после создания платежа запись попадает в `data/app-db.json -> payments` со статусом `pending`;
- после webhook `payment.succeeded` подписка активируется и продлевается;
- повторный webhook по тому же платежу не продлевает подписку второй раз (защита от дублей);
- фронт запрашивает `/api/subscription/status` и показывает: активна подписка / платеж в обработке.

Без ключей ЮKassa кнопки оплаты показывают понятную ошибку о том, что ключи не настроены.

## Kie.ai (Gemini) для нормализации карт
Чтобы после загрузки карта автоматически приводилась к удобной структуре:
- `KIE_API_KEY`
- `KIE_API_BASE` (по умолчанию `https://api.kie.ai`)
- `KIE_GEMINI_MODEL` (по умолчанию `gemini-2.5-pro`)
- `KIE_NORMALIZE_MAPS=true|false`

Для `PDF` используется локальный OCR первой страницы (`tesseract`) + Gemini, поэтому качество результата зависит от читаемости исходного PDF-изображения.

Если ключ не задан или AI временно недоступен, загрузка не ломается: сохраняется исходно распознанная карта.

## Структура
- `server.js` — backend API + сессии + подписки + роли.
- `index.html` — интерфейс.
- `styles.css` — стили.
- `script.js` — frontend логика (кабинет, тарифы, админка, карта).
- `data/app-db.json` — локальная база (пользователи, карты, платежи).
- `assets/` — изображения и видео.
